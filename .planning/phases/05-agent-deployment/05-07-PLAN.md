<task type="auto">
  <name>Task 1: Fix AgentLayer race condition with hex store initialization</name>
  <files>packages/client/src/scene/AgentLayer.tsx</files>
  <action>
Fix the race condition where old agents don't render at correct elevation because useMemo runs before hexStore is initialized.

**Root cause:** The useMemo at line 154 depends on [agents, getHexInfo] but getHexInfo returns null when called before hex store is populated. When agents load from server before HexWorld calls setHexData(), all elevations are calculated as 0.

**Solution:**
1. Import `isInitialized` from useHexStore:
   ```typescript
   const { getHexInfo, isInitialized } = useHexStore();
   ```

2. Add `isInitialized` to the useMemo dependency array (line 179):
   ```typescript
   }, [agents, getHexInfo, isInitialized]);
   ```

This forces the useMemo to recalculate agent positions when the hex store becomes initialized, ensuring correct elevation for agents that loaded before terrain data was ready.
  </action>
  <verify>
1. In browser console, verify no agents have y position of 0.02 (which indicates elevation=0 fallback)
2. Test: Reload page with existing agents - all should appear at correct hex elevations, not clustered at ground level
  </verify>
  <done>Old agents render at correct elevation immediately after page load, matching behavior of newly deployed agents</done>
</task>

<task type="auto">
  <name>Task 3: Increase HexTooltip readability at default zoom</name>
  <files>packages/client/src/scene/HexTooltip.tsx</files>
  <action>
Fix the tooltip being too small to read at default camera distance.

**Root cause:** Line 71 uses `distanceFactor={12}` which scales the tooltip based on camera distance. At the default zoom level, this makes the tooltip too small.

**Solution:**
Increase distanceFactor to reduce the scaling effect:

1. Change line 71 from:
   ```tsx
   distanceFactor={12}
   ```
   to:
   ```tsx
   distanceFactor={25}
   ```

A higher distanceFactor means the tooltip maintains more of its size as camera distance increases. Value of 25 keeps tooltip readable at default zoom while still providing some distance-based scaling.

Alternative considered: Using `transform={false}` to disable scaling entirely, but that causes tooltip to appear huge when zoomed in. The balanced approach with distanceFactor={25} works better.
  </action>
  <verify>
1. Hover over a hex at default zoom level (after page load, before any zoom)
2. Tooltip text should be clearly readable (biome name, coordinates, resource info)
3. Zoom in - tooltip should still look proportional (not oversized)
  </verify>
  <done>HexTooltip is readable at default zoom level while maintaining proportional scaling when zoomed</done>
</task>
