<task type="auto">
  <name>Task 1: Create Docker Compose configuration</name>
  <files>
    docker-compose.yml
  </files>
  <action>
Create docker-compose.yml at project root with:

1. PostgreSQL 16 Alpine service:
   - Image: postgres:16-alpine
   - Environment: POSTGRES_DB=landmind, POSTGRES_USER=landmind, POSTGRES_PASSWORD=landmind_dev
   - Port: 5432:5432
   - Named volume: postgres_data:/var/lib/postgresql/data
   - Healthcheck: pg_isready -U landmind (5s interval, 5 retries)

2. Redis 7 Alpine service:
   - Image: redis:7-alpine
   - Command: redis-server --appendonly yes
   - Port: 6379:6379
   - Named volume: redis_data:/data
   - Healthcheck: redis-cli ping (5s interval, 5 retries)

3. Define both named volumes at bottom of file

Use version '3.8' for compose file format.
  </action>
  <verify>
Run `docker compose config` to validate YAML syntax.
  </verify>
  <done>
docker-compose.yml exists and passes config validation.
  </done>
</task>

1. Start containers in detached mode:
   ```bash
   docker compose up -d
   ```

2. Wait for containers to be healthy (up to 30 seconds)

3. Verify PostgreSQL:
   ```bash
   docker compose exec postgres pg_isready -U landmind
   ```
   Expected: "accepting connections"

4. Verify Redis:
   ```bash
   docker compose exec redis redis-cli ping
   ```
   Expected: "PONG"

5. Test PostgreSQL connection from host:
   ```bash
   docker compose exec postgres psql -U landmind -d landmind -c "SELECT 1;"
   ```
   Expected: Returns "1"

If any check fails, check `docker compose logs` for errors and troubleshoot.
