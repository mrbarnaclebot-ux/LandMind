  <action>
Create PerformanceAdapter.tsx:
- Wrap scene content with `<PerformanceMonitor>` from @react-three/drei
- Track FPS and expose via context: `usePerformanceSettings()`
- State: { dpr, lodDistances, enableEffects, qualityLevel }
- On decline (FPS < 45): reduce dpr (min 0.5), shrink lodDistances, disable effects
- On incline (FPS > 55): increase dpr (max 2), expand lodDistances, enable effects
- Mobile detection: cap dpr at 1.5 on window.innerWidth < 768

Create ChunkedHexWorld.tsx:
- Props: gridRadius (default 500 for ~1M hexes), seed
- Initialize ChunkManager with generated hex data
- Use useFrame to:
  1. Get camera from useThree()
  2. Call chunkManager.getVisibleChunks() with camera frustum
  3. Call chunkManager.updateLODLevels() with camera position
- Render only visible chunks as BiomeChunk components
- Each BiomeChunk renders one InstancedMesh with geometry from LODHexGeometry[lodLevel]
- Dispose InstancedMesh when chunk goes out of view (memory management)

BiomeChunk component:
- Props: chunk, biome, geometry, hexes
- Memoize instance matrices
- Update instanceMatrix only when chunk becomes visible or LOD changes

Update ThreeScene.tsx:
- Replace `<HexWorld gridRadius={20} />` with `<ChunkedHexWorld gridRadius={500} />`
- Wrap with `<PerformanceAdapter>` component
- Keep gridRadius configurable via env var VITE_HEX_GRID_RADIUS (default 500)
