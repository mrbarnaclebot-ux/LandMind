Fix the snapPoints prop in BottomSheet.tsx:

1. The react-modal-sheet library requires snapPoints to be in ASCENDING order, starting from 0 (closed) and can go up to 1 (full height).

2. Change the default snapPoints from `[0.9, 0.5, 0.25]` (WRONG - descending) to `[0, 0.25, 0.5, 0.9]` (CORRECT - ascending with closed state).

3. Update initialSnap index:
   - Old: `initialSnap={1}` pointed to 0.5 (middle of descending array)
   - New: `initialSnap={2}` points to 0.5 (index 2 in ascending array [0, 0.25, 0.5, 0.9])
   - This opens the sheet at 50% height by default

4. The corrected code:
   ```tsx
   export function BottomSheet({
     isOpen,
     onClose,
     children,
     title,
     snapPoints = [0, 0.25, 0.5, 0.9],
   }: BottomSheetProps) {
     return (
       <Sheet
         isOpen={isOpen}
         onClose={onClose}
         snapPoints={snapPoints}
         initialSnap={2} // Start at 0.5 (50% height)
       >
   ```

5. Update the JSDoc comment for snapPoints prop to reflect correct format:
