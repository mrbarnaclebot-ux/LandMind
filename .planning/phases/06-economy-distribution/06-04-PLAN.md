  <name>Task 1: Create Merkle tree service</name>
  <files>packages/server/src/services/merkleService.ts</files>
  <action>
Create merkleService.ts for generating Merkle trees and proofs.

**Install dependency** (if not present): `npm install @noble/hashes` - provides keccak256 for Solana-compatible hashing.

**Types:**
```typescript
export interface UserShare {
  wallet: string;  // Base58 wallet pubkey
  claimableAmount: bigint;  // Lamports
}

export interface MerkleTreeResult {
  root: Uint8Array;
  tree: Map<string, Uint8Array[]>;  // wallet -> proof
  leaves: Uint8Array[];
}
```

**Functions:**

1. `hashLeaf(wallet: string, amount: bigint): Uint8Array`
   - Decode wallet from base58 to bytes
   - Create 8-byte buffer for amount (little-endian)
   - Concatenate and hash with keccak256
   - Use @noble/hashes keccak_256

2. `generateMerkleTree(shares: UserShare[]): MerkleTreeResult`
   - Generate leaves from shares using hashLeaf
   - Sort leaves for deterministic tree
   - Build tree bottom-up: hash pairs to create parent nodes
   - Store proof paths for each leaf
   - Return root, tree map, and leaves

3. `generateProof(wallet: string, amount: bigint, tree: MerkleTreeResult): Uint8Array[] | null`
   - Look up wallet in tree map
   - Return proof array or null if not found

4. `verifyProof(proof: Uint8Array[], root: Uint8Array, leaf: Uint8Array): boolean`
   - Recompute root from leaf using proof
   - Compare with expected root (same algorithm as smart contract)

**Implementation notes:**
- Use @noble/hashes for keccak256 (same as solana_program::keccak)
- Use bs58 for base58 decoding
- Sort leaves before building tree for deterministic roots
- Handle odd number of leaves by duplicating last leaf

```typescript
import { keccak_256 } from '@noble/hashes/sha3';
import bs58 from 'bs58';
```

  <files>packages/server/src/services/feeMonitor.ts</files>
  <action>
Create feeMonitor.ts to track incoming fee deposits:

**Environment variables:**
- PUMPFUN_FEE_WALLET: Wallet that receives PumpFun fees (optional, skip monitoring if not set)

**Functions:**

1. `async checkForFeeDeposits(connection: Connection): Promise<number>`
   - Get recent signatures for PUMPFUN_FEE_WALLET
   - Filter to deposits not yet recorded in database
   - Parse transactions to extract deposit amounts
   - Record each new deposit as FeeDeposit with source: 'PUMPFUN'
   - Return count of new deposits

2. `async checkTreasuryDeposits(connection: Connection): Promise<number>`
   - Get recent signatures for treasury PDA
   - Filter to new deposits (not from claims)
   - Record as FeeDeposit with source: 'DEPLOYMENT'
   - Return count of new deposits

3. `startFeeMonitor(connection: Connection): void`
   - Start interval timer (60 seconds)
   - Call checkForFeeDeposits and checkTreasuryDeposits
   - Log results
   - Store last processed slot to avoid reprocessing

4. `async getTotalFeePool(): Promise<bigint>`
   - Query sum of FeeDeposit amounts where processed = false
   - Or query treasury balance directly

**Implementation:**
```typescript

  : null;
```
