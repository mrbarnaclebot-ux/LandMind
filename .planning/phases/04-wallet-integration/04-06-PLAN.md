  <files>packages/client/src/lib/transactions.ts</files>
  <action>
Add a generic retry helper and wrap RPC calls with it:

1. Create `retryWithBackoff<T>` helper function at top of file:
   - Parameters: `fn: () => Promise<T>`, `maxRetries: number = 3`, `baseDelayMs: number = 1000`
   - Retry loop with exponential delays: 1s, 2s, 4s (baseDelayMs * 2^attempt)
   - On each failure, check if error is retryable (rate limit, network errors)
   - After all retries exhausted, throw enriched error with `code` property

2. Create error classification helper `classifyRpcError(error: unknown)`:
   - Returns object: `{ code: 'RATE_LIMIT' | 'NETWORK' | 'NO_DATA' | 'UNKNOWN', retryable: boolean, message: string }`
   - Rate limit detection: error message contains "429", "rate limit", "too many requests" (case insensitive)
   - Network detection: error message contains "fetch", "network", "ECONNREFUSED", "timeout"
   - NO_DATA: not an error, just empty results (handle in main function)
   - Default to UNKNOWN with retryable: false

3. Create custom error class `TransactionFetchError extends Error`:
   - Properties: `code`, `retryable`, `originalError`

4. Modify `fetchTransactionHistory` to:
   - Wrap `connection.getSignaturesForAddress()` call in `retryWithBackoff()`
   - Wrap `connection.getParsedTransactions()` call in `retryWithBackoff()`
   - If both calls succeed but signatures.length === 0, return empty array (not an error)
   - On failure after retries, throw `TransactionFetchError` with classified code

Keep all existing parsing logic unchanged. Only wrap the two RPC calls.
