  <files>packages/client/src/hex/hexMath.ts</files>
  <action>
Create hexMath.ts with flat-top axial coordinate system:

1. Define constants:
   - HEX_SIZE = 1.0 (outer radius)
   - ELEVATION_STEP = 0.5 (height per elevation tier)

2. Implement coordinate conversions (flat-top orientation):
   - hexToPixel(q, r) -> { x, z }: Convert axial to world position
     - x = HEX_SIZE * (3/2 * q)
     - z = HEX_SIZE * (sqrt(3)/2 * q + sqrt(3) * r)
   - pixelToHex(x, z) -> { q, r }: Convert world to axial (with rounding)

3. Implement hex utilities:
   - hexDistance(a, b): Manhattan distance in axial coords
   - hexNeighbors(q, r): Return 6 neighbor coordinates
   - hexRound(q, r): Round fractional axial coords to nearest hex

4. Export HexCoord type: { q: number; r: number }

Use the formulas from Red Blob Games (documented in RESEARCH.md). Flat-top orientation means hex corners are at angles 0, 60, 120, 180, 240, 300 degrees.
  </action>

<task type="auto">
  <name>Task 2: Create beveled hex mesh generator</name>
  <files>packages/client/src/hex/hexMesh.ts</files>
  <action>
Create hexMesh.ts with procedural beveled hex mesh generator:

1. Import from @babylonjs/core: Mesh, VertexData, Vector3, Scene

2. Implement createBeveledHexMesh(scene, options):
   - options: { size?: number, height?: number, bevelSize?: number }
   - Defaults: size=1.0, height=0.3, bevelSize=0.08

3. Build geometry (flat-top orientation):
   - Calculate 6 outer corners at angles 0, 60, 120... degrees
   - Calculate 6 inner corners (scaled by (size - bevelSize) / size)
   - Build vertices:
     - Index 0: Top center (0, height, 0)
     - Indices 1-6: Top inner ring (at height)
     - Indices 7-12: Top outer ring (at height - bevelSize*0.5 for bevel)
     - Indices 13-18: Bottom outer ring (at y=0)
   - Build triangles:
     - Top face: 6 triangles from center to inner ring
     - Bevel face: 12 triangles (quads) inner ring to outer ring
     - Side faces: 12 triangles (quads) outer top to bottom

4. Create VertexData, compute normals, apply to mesh

5. Return the mesh (not added to scene yet - template for thin instances)

Use counter-clockwise winding for proper face normals. Call VertexData.ComputeNormals after setting positions/indices.
  </action>
